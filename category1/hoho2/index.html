npm run build

npm run build를 하면 dist폴더가 생깁니다.
dist폴더 하위에 js, css 그리고 index.html, favicon.ico 요롷게 생긴다.

npm run build로 생성된 dist 폴더구조

dist 
  └ css
    └ app.~~~.css
  └ js
    ├ app.~~~.js
    ├ app.~~~.js.map
    ├ chunk-vendors.~~~.js
    └ chunk-vendors.~~~.js.map
  ├ favicon.ico
  └ index.html

index.html 내부 link태그의 href속성과 script태그의 src 속성에 /js 또는 /css 로 시작하는 주소를 자신의 repository 이름으로 시작해야 합니다!

/v-todo/js… , /v-todo/css…

그리고 "file":" 로 전체 검색을 하면, .js.map 파일 내부에서 참조하는 파일 주소가 작성되어 있는데 그것 역시 자신의 레포 주소로 시작하면 됩니다!

js/~~ // 이렇게 되어있는 주소들을
v-todo/js // 이렇게 자신의 repository 이름으로 시작하게 합니다.
/js/~~ // 슬래시가 있던 친구들은 그대로 슬래시를 처음에 두세요!
/v-todo/js // 이렇게요!

gh-pages 에 배포하기

$ git add dist/
$ git commit -m'프로젝트 배포!'
$ git subtree push --prefix dist origin gh-pages

git subtree 라는 명령어가 지정한 dist 폴더 내부의 파일들을 gh-pages 브랜치에 올려주는 작업을 진행합니다.
